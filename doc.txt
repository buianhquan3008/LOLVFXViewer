# Project: LoL Asset Studio

Build a Windows desktop application called **LoL Asset Studio**.

The goal is to create a standalone asset preview and iteration tool for League of Legends modding research. The tool must preview extracted assets outside the game. It must not inject into the game, modify the running League client, automate gameplay, bypass anti-cheat, or provide any competitive advantage.

This is a **development and visualization tool**, not a game engine, not a mod loader, and not a mod injector.

---

## Core Goal

Create a desktop application that can:

1. Browse an extracted League of Legends asset folder.
2. Preview textures.
3. Display 3D character models.
4. Display skeletons and bone hierarchy.
5. Play skeletal animations.
6. Parse Riot BIN text exports.
7. Build an internal VFX graph from BIN data.
8. Preview simplified VFX using a custom particle runtime.
9. Attach VFX to animated bones.
10. Hot reload modified text BIN files for faster iteration.

The first deliverable should be a working MVP, not a full final product.

---

## Important Scope Rules

Do not implement:

* Game injection
* Runtime memory editing
* Anti-cheat bypass
* Online gameplay modifications
* Riot client automation
* Direct patching of the live League installation

Only support:

* User-selected local extracted folders
* Text BIN exports
* Local preview rendering
* Developer inspection tools

## Coding Requirements

Use:

* Modern C++20
* RAII
* Smart pointers
* `std::filesystem`
* `std::optional`
* `std::variant` where useful
* Clear ownership rules
* No global mutable state
* Clear error handling
* Detailed comments for complex parsing/rendering logic
* CMake targets for app and tests

Avoid:

* Giant single-file implementation
* Hardcoded absolute paths
* Hidden global OpenGL state without wrapper classes
* Fake parsers that silently ignore errors
* Placeholder code that pretends to support Riot formats without clear TODO notes

# Phase 3: BIN Text Parser

Implement a parser for Riot BIN text exports, not binary BIN files at first.

Goal:

* Parse text-exported BIN files into a structured AST.
* Preserve unknown fields.
* Report parse errors with line and column.
* Do not crash on unknown types.

Support important object types gradually:

```text
SkinCharacterDataProperties
ResourceResolver
VfxSystemDefinitionData
VfxEmitterDefinitionData
VfxParticleDefinitionData
Materials
Textures
Curves
Child effects
Attachment/bone references
```

Parser requirements:

* Tokenizer
* AST nodes
* Type name
* Field name
* Primitive values
* Strings
* Numbers
* Booleans
* Lists
* Objects
* Hash/reference-like fields
* Unknown field preservation

Create tests for:

* Empty file
* Simple object
* Nested object
* List of objects
* Unknown field
* Invalid syntax
* VFX system with emitters
* Curves

Output:

```cpp
struct BinObject
{
    std::string typeName;
    std::unordered_map<std::string, BinValue> fields;
};

using BinValue = std::variant<
    std::monostate,
    bool,
    int64_t,
    double,
    std::string,
    std::vector<BinValue>,
    BinObject
>;
```

# Phase 4: VFX Graph

Convert parsed BIN AST into an internal VFX graph.

Do not try to support every Riot feature immediately.

Internal representation:

```text
VfxGraph
├── VfxSystem
│   ├── Emitters
│   ├── Materials
│   ├── Curves
│   ├── ChildSystems
│   └── Attachments
```

Suggested types:

```cpp
struct VfxGraph
{
    std::string name;
    std::vector<VfxEmitter> emitters;
    std::vector<VfxMaterialRef> materials;
    std::vector<VfxCurve> curves;
    std::vector<VfxChildEffect> children;
};

struct VfxEmitter
{
    std::string name;
    float spawnRate = 0.0f;
    float lifetimeMin = 1.0f;
    float lifetimeMax = 1.0f;
    glm::vec3 velocityMin = glm::vec3(0);
    glm::vec3 velocityMax = glm::vec3(0);
    glm::vec4 startColor = glm::vec4(1);
    glm::vec4 endColor = glm::vec4(1, 1, 1, 0);
    float startSize = 1.0f;
    float endSize = 0.0f;
    std::string texturePath;
    std::string materialName;
};
```

Mapping approach:

* Map known BIN fields to internal fields.
* Unknown fields should appear in the Inspector.
* Log unsupported features clearly.
* Prefer partial preview over failure.

# Phase 5: Particle Runtime

Create a custom CPU particle runtime first.

Particle data:

```cpp
struct Particle
{
    glm::vec3 position;
    glm::vec3 velocity;
    glm::vec4 color;
    float size;
    float rotation;
    float age;
    float lifetime;
};
```

Emitter features:

* Spawn rate
* Burst spawn
* Lifetime min/max
* Velocity min/max
* Color over lifetime
* Size over lifetime
* Rotation over lifetime
* Billboard rendering
* Texture animation placeholder
* Max particle count

Renderer:

* Use OpenGL instancing
* Billboard quads
* Texture atlas support placeholder
* Additive and alpha blend material modes
* Depth sorting can be added later

Required for first particle version:

* Spawn particles
* Update particles
* Kill expired particles
* Render billboard quads
* Draw particle count in Inspector

# Phase 6: Champion + VFX Integration

Attach VFX to bones.

Features:

* Select bone from skeleton hierarchy
* Attach VFX system to bone
* Effect follows animated bone transform
* Support multiple active effects
* Show attachment list in Inspector

Common bone names:

```text
root
head
weapon
hand_l
hand_r
wing
spine
```

Implementation idea:

```cpp
struct VfxAttachment
{
    std::string boneName;
    glm::mat4 localOffset;
    std::shared_ptr<VfxSystemInstance> vfx;
};
```

Each frame:

```text
1. Animator updates skeleton pose.
2. Bone world matrices are computed.
3. VFX attachment reads target bone matrix.
4. VFX emitter transform is updated.
5. Particle runtime updates and renders.
```

# Phase 7: Hot Reload

Watch text BIN files.

When modified:

1. Detect file change.
2. Reparse BIN.
3. Rebuild VFX graph.
4. Preserve UI selection if possible.
5. Update active VFX preview.
6. Log success or parse error.
7. Do not crash on invalid file.

Use a simple polling file watcher first:

```text
Every 0.5 seconds:
- Check last_write_time
- If changed, reload
```

Later it can be replaced with Win32 file watching.
